# AgentLab

AgentLab is a local Windows workflow for queueing, running, and reviewing Codex-powered app builds. It combines markdown queue files, role-specific agent prompts, PowerShell orchestration scripts, and a local React/Express console.

The repository is meant to be cloned as the AgentLab runtime. Apps generated by AgentLab live under `apps\` and are ignored by this root repository by default so each generated app can keep its own Git history.

## Prerequisites

- Windows with PowerShell
- Git
- Codex CLI installed, signed in, and available as `codex`
- Node.js 20, 22, or 24+ and npm, if you want to run the local console

## Clone

```powershell
git clone https://github.com/chant-dev/agent-lab.git
cd agent-lab
```

The PowerShell scripts infer the AgentLab root from their own location. You can still pass `-Root` explicitly when needed:

```powershell
.\agent-runtime\scripts\check-agentlab.ps1 -Root (Resolve-Path .)
```

## Health Check

Run this first after cloning:

```powershell
.\agent-runtime\scripts\check-agentlab.ps1
```

The check verifies the required queue files, prompt files, runtime scripts, docs, and common command-line tools.

## Queue An App Idea

```powershell
.\agent-runtime\scripts\add-idea-to-queue.ps1 `
  -Idea "Build a local desktop app that tracks project habits and exports weekly summaries" `
  -BuildMode standard
```

This runs the Queue Item Generator prompt through Codex and appends a `[READY]` item to `queue\queue.md`.

## Run The Next Queue Item

```powershell
.\agent-runtime\scripts\run-next-queue-item.ps1 -Mode standard
```

Modes:

- `lightweight`: Scaffolder, Builder, Tester, Reviewer
- `standard`: Scaffolder, Builder, Tester, Refiner, Tester, Reviewer, Deployer, Reviewer
- `production`: Standard flow with an additional Refiner and Tester pass

Generated apps are created under `apps\app-name`. They are intentionally ignored by the root repo.

## Local Console

The console is a local-only control panel for queue intake, queue state, health checks, command output, app folders, reports, and runtime paths.

```powershell
cd agent-runtime\console
npm install
npm run dev
```

By default, the API binds to `127.0.0.1:8787`; the Vite client runs on its own local dev server and proxies API calls.

For a production-style local run:

```powershell
npm run build
npm start
```

## Repository Layout

```txt
queue\                    Active, completed, and failed queue markdown files
apps\                     Generated app repositories, ignored by this root repo
agent-runtime\prompts\    Role prompts used by the Codex agent passes
agent-runtime\scripts\    PowerShell orchestration scripts
agent-runtime\docs\       Runbook and parallel-agent guidance
agent-runtime\console\    Local React/Express control console
```

## Git And Secrets

Do not commit `.env`, tokens, API keys, local databases, generated app dependencies, or runtime logs. The root `.gitignore` keeps generated app folders, logs, worktrees, Node artifacts, Python virtual environments, and SQLite runtime files out of the AgentLab repository.

If you want to publish a generated app, push that app folder as its own repository.
